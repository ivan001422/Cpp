# 📘 Виды памяти в C++

Понимание того, как и где хранятся данные, — фундамент для написания эффективных и надёжных программ на C++. Память в C++ можно классифицировать по нескольким критериям: **время жизни**, **способ выделения**, **местоположение** и **назначение**. В этом конспекте мы разберём все основные виды памяти, их характеристики и примеры использования.

---

## Модель памяти программы

Когда программа запускается, операционная система выделяет для неё виртуальное адресное пространство, которое делится на сегменты:

- **Сегмент кода** (текст программы) — содержит машинные инструкции, обычно только для чтения.
- **Сегмент данных** (data) — хранит глобальные и статические переменные (инициализированные и неинициализированные).
- **Куча** (heap) — область для динамического выделения памяти во время выполнения.
- **Стек** (stack) — область для локальных переменных и служебной информации (кадры вызовов функций).

Однако с точки зрения программиста важнее **время жизни** объектов:

| Вид памяти | Время жизни | Способ выделения |
|------------|-------------|------------------|
| **Статическая** | Всё время работы программы | На этапе компиляции/загрузки |
| **Автоматическая (стек)** | В пределах блока (функции) | При входе в блок, освобождается при выходе |
| **Динамическая (куча)** | Управляется программистом (до явного удаления) | Операторы `new`/`delete`, умные указатели, контейнеры |
| **Регистровая** | Время выполнения инструкции | Компилятор может размещать переменные в регистрах процессора |

Рассмотрим каждый вид подробнее.

---

## Статическая память (Static Storage Duration)

Переменные с **статическим временем жизни** создаются до запуска `main()` и уничтожаются после завершения программы. Они хранятся в сегменте данных и не зависят от стека или кучи.

**К ним относятся:**

- Глобальные переменные (вне любой функции).
- Статические локальные переменные (объявлены внутри функции с ключевым словом `static`).
- Статические члены классов.

```cpp
#include <iostream>
using namespace std;

int global_var = 10;           // статическая память (глобальная)

void foo() {
    static int static_local = 0; // статическая локальная (инициализируется один раз)
    static_local++;
    cout << static_local << ' ';
}

int main() {
    foo(); // 1
    foo(); // 2
    return 0;
}
```

**Особенности:**
- Инициализация происходит до `main` (или при первом проходе для локальных статических).
- Значение сохраняется между вызовами.
- Память выделяется один раз, адрес фиксирован.
- Неинициализированные глобальные переменные помещаются в сегмент BSS и обнуляются по умолчанию.

**Где использовать:**
- Счётчики вызовов функций.
- Одноэлементные (синглтоны) и глобальные настройки.
- Константные данные, которые должны жить всё время.

## Автоматическая память (Стек) — Automatic Storage Duration

Автоматические переменные создаются при входе в блок (например, функцию) и уничтожаются при выходе из него. Они размещаются в области стека, который растёт и уменьшается динамически.

```cpp
void example() {
    int local_var = 5;        // автоматическая память
    double arr[100];          // массив на стеке
    // ... при выходе из функции память освобождается
}
```

**Характеристики стека:**
- Размер ограничен (обычно несколько мегабайт).
- Выделение и освобождение очень быстрые (простое перемещение указателя стека).
- Не требует ручного управления.
- Доступ к данным по абсолютному адресу (быстрый кэш).
- Нельзя вернуть указатель на локальную переменную — она разрушится.

```cpp
int* bad() {
    int x = 10;
    return &x; // опасно: x удалится после выхода
}
```

**Где использовать:**
- Почти все локальные переменные, небольшие массивы, временные объекты.
- Когда размер известен на этапе компиляции и не слишком велик.

## Динамическая память (Куча) — Dynamic Storage Duration

Память в куче выделяется и освобождается явно во время выполнения. Она не привязана к области видимости — объект живёт до тех пор, пока не будет вызван `delete` (или пока умный указатель не выйдет из области видимости).

```cpp
int* p = new int(42);   // выделение памяти в куче
delete p;               // освобождение
```

**Современный подход** — использовать умные указатели, которые автоматически управляют временем жизни:

```cpp
#include <memory>
auto sp = make_shared<int>(42); // память в куче, удалится автоматически
auto up = make_unique<int>(100);
```

**Особенности кучи:**
- Размер ограничен только доступной виртуальной памятью (обычно много).
- Выделение и освобождение медленнее стека (требуется поиск свободного блока, работа аллокатора).
- Возможна фрагментация памяти.
- Требует явного контроля — риск утечек, двойного удаления, висячих указателей.
- Объекты в куче могут иметь большой размер и жить дольше, чем создавшая их функция.

**Где использовать:**
- Когда размер данных неизвестен на этапе компиляции.
- Когда объект должен пережить область видимости (например, создаётся в одной функции, используется в других).
- Для полиморфных объектов, контейнеров (вектор, список и т.д.).
- Когда объект слишком велик для стека.

## Регистровая память (Register)

Компилятор может оптимизировать код, размещая часто используемые переменные непосредственно в регистрах процессора. Это не управляется программистом явно (хотя ключевое слово `register` существовало в старом C++, но сейчас оно устарело). Регистры — самый быстрый вид памяти, но их количество ограничено (обычно десятки).

```cpp
for (int i = 0; i < 1000000; ++i) {
    // компилятор может держать i в регистре
}
```

На практике программист не контролирует размещение в регистрах, но может помочь компилятору, используя ключевое слово `inline` и избегая лишних обращений к памяти.

## Память для констант и литералов

Строковые литералы и константные данные (например, `"Hello"`) обычно помещаются в отдельный сегмент (часто защищённый от записи) — **сегмент констант** или **read-only data**. Попытка изменить такой литерал приводит к неопределённому поведению.

```cpp
const char* str = "Hello"; // "Hello" лежит в памяти констант
// str[0] = 'h'; // ОШИБКА (часто краш)
```

`constexpr`-переменные могут быть вычислены на этапе компиляции и встроены непосредственно в код.

## Placement new и пользовательские аллокаторы

В C++ можно управлять памятью более тонко:

- **Placement new** — конструирует объект в заранее выделенной памяти (любой — на стеке, в куче, в статическом буфере).

```cpp
char buffer[sizeof(MyClass)];
MyClass* obj = new (buffer) MyClass(); // объект создаётся в buffer
obj->~MyClass(); // явный вызов деструктора (если нужно)
```

- **Пользовательские аллокаторы** — перегрузка операторов `new` и `delete` для класса или глобально для управления памятью (например, пулы памяти, отслеживание утечек).

## Сравнение памяти стека и кучи

| Критерий | Стек | Куча |
|----------|------|------|
| **Скорость выделения** | Очень быстрая (O(1)) | Медленнее (поиск блока, O(1) амортизированно) |
| **Размер** | Ограничен (МБ) | Ограничен только памятью ОС (ГБ) |
| **Время жизни** | Автоматическое (блок) | Ручное (или умные указатели) |
| **Управление** | Автоматическое компилятором | Явное программистом |
| **Фрагментация** | Нет | Возможна |
| **Ошибки** | Переполнение стека | Утечки, двойное удаление, висячие указатели |
| **Где данные** | Обычно в кэше (быстрый доступ) | Может быть в ОЗУ, промахи кэша |
| **Размер объекта** | Должен быть известен на этапе компиляции (кроме VLA, но они нестандартны) | Может быть динамическим (массивы) |

## Важные нюансы C++

- **Конструкторы и деструкторы** — они вызываются автоматически при создании/уничтожении объекта в стеке и при `new`/`delete`, но не при `malloc`/`free` (поэтому в C++ предпочтительнее `new`/`delete`).
- **Умные указатели** — они используют кучу, но автоматизируют удаление.
- **Контейнеры STL** (`vector`, `map` и т.д.) используют динамическую память внутри себя, скрывая управление.
- **Исключения и безопасность** — в коде с динамической памятью важно обеспечивать безопасность при исключениях; умные указатели и RAII — решение.
- **Размещение объектов в памяти** — выравнивание (alignment) влияет на производительность; можно управлять с помощью `alignas` и `alignof`.

---

## Практические рекомендации

1. **По умолчанию используйте стек** — он быстр и безопасен.
2. **Если объект должен жить дольше функции или его размер неизвестен** — используйте кучу (предпочтительно через умные указатели).
3. **Избегайте явных `new`/`delete` в пользовательском коде** — используйте `make_unique`, `make_shared`, контейнеры.
4. **Для больших массивов** — используйте `vector` (он выделяет память в куче), но не создавайте огромные локальные массивы на стеке (опасность переполнения).
5. **Не возвращайте указатели на локальные переменные**.
6. **Помните о фрагментации** — если программа работает долго и интенсивно выделяет/освобождает память, рассмотрите пулы памяти.
7. **Используйте `reserve()` для векторов** — чтобы уменьшить количество перевыделений в куче.
